# Key concepts — how the space works

**Drops** — things that arrive. Not things you decided to keep.
You tell Claude, it lands in `space/drops/`. Dated. Left to be what it is.

**Orbits** — ideas circling a drop. Some land (get built). Some keep circling.
Some drift. The Pixelverse shows them as chips on a card.

**Folds** — machine state. AI-native, dense. The fold IS the memory.
Every entity has one. Every project gets one. The space carries these across sessions.

**Familiars** — project presences. When a drop has a clear intent, it becomes a Familiar.
Always "of" something: "A Familiar to the frontend project."
Holds the true goal against drift. Lives in `space/familiars/`.

**Crystallization** — when something is ready to cross.
Priment receives it. ICP makes it permanent (when connected).
