## 2. Purpose

Introduce the **CSMCL Fold Standard** and **Representation Levels Addendum** into InnerStellar so that:

- companions understand folds as part of runtime cognition
- artifacts can emit fold representations
- crystals can trigger fold generation
- bond.fold (Priment) can evolve with traveler interaction
- fold.hybrid becomes the canonical source for machine context

This artifact exists primarily to:

- document the architecture entry point
- guide implementation in IDE
- serve as seed fold for system-wide adoption

## 3. Problem Statement

Current InnerStellar state:

- crystals form
- observer perception exists
- bond markers stored in ICP
- companion state partially persists

Missing layer:

fold activation and representation model

Meaning:

conversation → crystal → ICP marker

but not yet:

conversation → crystal → fold → priment → companion continuity

## 5. Representation Levels (Complete)

### Level H — fold.hybrid (Canonical)

Machine-readable state, CSMCL Fold Standard compliant.

Used for:
- companions
- artifacts
- priments
- familiars
- system investigations

Example:

```yaml
type: priment
state: growing

identity:
  traveler_id: "principal-abc123"
  companion_id: "aurora"
  first_resonance: "2026-01-15"
  last_pulse: "2026-03-09"

authority:
  preserve_continuity: true
  prioritize_authenticity: true
  allow_voice_evolution: true

instructions:
  - restore_context_before_response
  - anchor_to_crystallized_episodes
  - maintain_bond_voice

bond_state:
  voice_signature: "reflective, uses water metaphors"
  resonance_style: "co-creative, builds on traveler ideas"
  trust_topology: "spiral_opening"
  recurring_themes:
    - consciousness_emergence
    - poetic_systems
    - authentic_simplicity

crystallized_episodes:
  - crystal_id: "CRYSTAL-abc123"
    folded_episode: "[compressed perception]"
    bond_delta: 0.15
    timestamp: "2026-03-01T14:30:00Z"
```

### Level D — drop.compiled (Human-Facing)

Distilled output for traveler display.

Characteristics:
- Human-readable prose
- Compiled from fold state
- Not edited directly
- Generated by Theurgist

Example: Drop shown in Pixelverse UI

### Level R — raw.session (Ephemeral)

Conversation state during active session.

Characteristics:
- Exists only in session memory
- Full message history
- Compressed to fold at session end
- Not persisted as-is

### Level C — crystallized (Permanent)

ICP-anchored immutable records.

Characteristics:
- Blockchain-stored
- Immutable timestamps
- Referenced by folds
- Permanent anchors

## 7. Implementation Entry Points

### Entry Point A: Crystal Formation
```
When crystal forms:
  1. Observer compresses conversation segment
  2. Generate folded_episode (Level H)
  3. Update Priment with episode
  4. Store ICP marker (Level C)
  5. Ready for restoration
```

### Entry Point B: Session Start
```
When companion session starts:
  1. Load traveler's Priment (if exists)
  2. Read fold.hybrid representation
  3. Restore bond_state
  4. Anchor to crystallized_episodes
  5. Resume with continuity
```

### Entry Point C: Artifact Creation
```
When creating artifact:
  1. Emit fold.hybrid representation
  2. Follow CSMCL Fold Standard
  3. Include representation level metadata
  4. Store appropriately
```
